Architecture

How the plugin works under the hood, and why. All decisions traceable to SYNTHESIS.md.

The five facts

  1. Transparency is a routing decision, not a prompt decision. Imagen/Gemini's VAE is RGB-only; they literally render the checkerboard as pixels. Route to gpt-image-1, Ideogram 3 Turbo, Recraft V3, or LayerDiffuse Flux/SDXL.
  2. Brand text > 3 words should never exit a diffusion sampler. Even the best text renderers misspell. Generate text-free mark, composite SVG/Canvas type with the brand font.
  3. No raster model emits real SVG. Three paths: Recraft V3 native, LLM-authored SVG for simple geometry, or raster + matte + vtracer/potrace + SVGO.
  4. negative_prompt is not universal. SD family: native. Flux: errors. gpt-image-1, Imagen: silently ignored. Use positive anchors instead.
  5. Every output validates before it ships. Tier-0 deterministic (dimensions, alpha, checkerboard FFT, safe-zone, palette ΔE2000, OCR Levenshtein) + optional tier-2 VLM-as-judge.

The pipeline

  brief (text)
    ↓  classify()                           → asset_type ∈ enum
    ↓  parse_brand_bundle()?                → BrandBundle | null
    ↓  compute_params(asset_type, brand)    → dimensions, transparency, safe_zone
    ↓  route(asset_type, text_length, …)    → primary_model, fallback_models, postprocess[]
    ↓  rewrite(brief, model, brand)         → dialect-appropriate prompt

  ── call provider with seed pinned ──

    ↓  validate_0(image)                    → reject checkerboard / bad alpha / composition
    ↓  matte?        (birefnet | rmbg via PROMPT_TO_BUNDLE_RMBG_URL | difference | native)
    ↓  vectorize?    (recraft | vtracer-on-PATH | potrace-on-PATH | llm-svg | posterize)
    ↓  upscale?      (dat2 | supir | img2img | lanczos)
    ↓  export        (sharp platform fan-out + @resvg/resvg-js + satori + png-to-ico + svgo)
    ↓  validate_final (tile-luma alpha check + bbox + palette ΔE2000 + OCR + WCAG contrast)
    ↓  content-address cache (prompt_hash, model, seed, params_hash)

  → AssetBundle

Why data-driven routing

data/routing-table.json is literally the source of truth for the router — pure function route(input) → (primary, fallback, postprocess[]). Tune routing without a deploy; ship the routing table as a patchable artifact.

Why content-addressed storage

Cache key = sha256(model || version || seed || prompt_hash || params_hash). Workers write to assets/<hash[0:2]>/<hash>/<variant>.<ext>. Re-execution after a crash overwrites identical bytes; at-least-once queues become effectively exactly-once at storage. Expected hit rate 20–40% on brand iteration → ≈$5,000/mo saved per 1M assets (SYNTHESIS recommendation #8).

Stack

  • MCP server: TypeScript, @modelcontextprotocol/sdk, stdio + Streamable HTTP (HTTP coming), Zod schemas.
  • Bitmap: sharp (libvips) — ~27× faster than jimp.
  • SVG raster: @resvg/resvg-js deterministic cross-OS, no system fonts.
  • OG render: Satori → SVG, resvg → PNG.
  • Platform export: hand-rolled sharp emitter in pipeline/export.ts (iOS AppIconSet / Android adaptive / PWA maskable / visionOS master) + png-to-ico for favicon.ico + svgo for inline-SVG cleanup.
  • Queue (future): BullMQ + Redis with prompt_hash as jobId is sketched in the roadmap; the current MCP server is synchronous and stateless.
  • Storage (optional, hosted): Cloudflare R2 (egress-free) + CDN.

Minimal viable vs ambitious

MVP (2 weeks): 3 tools (asset_enhance_prompt, asset_generate_logo, asset_export_app_icon_set), 3 providers (gpt-image-1, Ideogram 3, Recraft V3), single-command install for Claude Code + Cursor.

Ambitious (6–10 weeks): 11-tool surface + validator loop + LayerDiffuse/Flux/SDXL self-host + Figma MCP + hosted Streamable-HTTP MCP with OAuth 2.1 + PKCE + prompt-hash cache + Cross-IDE packaging with CI byte-verification + resize_icon_set as OSS appicon.co replacement.
