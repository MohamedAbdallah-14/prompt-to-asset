Install

Prerequisites

  • Node.js ≥ 20.11
  • Optional: sharp (auto-installed on supported platforms)
  • At least one provider API key (see providers)

Clone & build

git clone https://github.com/yourorg/prompt-to-asset.git
cd prompt-to-asset
npm install
npm run build
bash scripts/sync-mirrors.sh

Claude Code

The sync script writes .claude-plugin/plugin.json and copies every skill into .claude/skills/. Claude Code picks them up automatically when you /plugin install or when the repo is opened.

Cursor

After bash scripts/sync-mirrors.sh, you get:

  • .cursor/rules/prompt-to-asset.mdc — always-on asset-routing rule
  • .cursor/skills/{asset-enhancer,logo,app-icon,...}/SKILL.md — the full skill bundle
  • .cursor/mcp.json — MCP server registration pointing at the compiled binary

Restart Cursor; new chats will have all 11 asset_* tools available.

Codex CLI

AGENTS.md and .codex/config.toml are regenerated by the sync script. Codex reads both automatically.

Gemini CLI

gemini-extension.json + GEMINI.md are regenerated. Gemini CLI merges them into its context.

GitHub Copilot

.github/copilot-instructions.md is the always-on rule. .github/copilot-skillset.json exposes a single asset_generate(asset_type) skill to stay under Copilot's 5-skill cap.

Windsurf & Cline

.windsurf/rules/prompt-to-asset.md (activation: always_on) and .clinerules/01-prompt-to-asset.md.

VS Code MCP

.vscode/mcp.json registers the server for VS Code's MCP support.

Your first call

Any of these briefs trigger the asset-enhancer rule and route through the MCP tools:

> Generate a transparent logo for my app "Acme" — simple, modern, blue and white
> Create an app icon of a rocket for my iOS + Android + PWA
> I need a favicon bundle with dark-mode support
> Make an OG image titled "Introducing Acme" with subtitle "The fastest framework in JS"
> Convert this PNG to SVG: /path/to/logo.png

Under the hood, the agent calls asset_enhance_prompt first to get a structured AssetSpec, then invokes the relevant asset_generate_* tool.

Using a brand bundle

Drop a brand.json, DTCG tokens.json, or a brand.md with hex colors in your repo. Any tool accepts a brand_bundle arg; the asset_brand_bundle_parse tool canonicalizes it.

// brand.json
{
  "palette": ["#0f172a", "#38bdf8", "#ffffff"],
  "style_refs": ["./brand/style-01.png", "./brand/style-02.png"],
  "do_not": ["drop shadows", "lens flare", "gradient sky"],
  "typography": { "primary": "Inter", "secondary": "JetBrains Mono" },
  "logo_mark": "./brand/logo-mark.svg"
}

The rewriter layers palette (as exact hex enumeration), style refs (as IP-Adapter / --sref / Recraft style_id), and the do_not list (rewritten as positive anchors — "no shadows" becomes "flat matte surfaces") into every generation call. Palette adherence is validated by K-means + ΔE2000 on the output; drift > 10 triggers regeneration.

Configuring provider API keys

Set whichever you have; the router will use whatever is available and fall back gracefully.

export OPENAI_API_KEY=sk-...           # gpt-image-1, gpt-image-1.5, dall-e-3
export GEMINI_API_KEY=...              # Imagen 3/4, Gemini Flash Image
export IDEOGRAM_API_KEY=...            # Ideogram 2/2a/3/3-turbo
export RECRAFT_API_KEY=...             # Recraft V2/V3/V4 (native SVG)
export BFL_API_KEY=...                 # Flux Pro / 1 / 2 / Kontext

Optional — self-hosted post-processing endpoints:

export PROMPT_TO_BUNDLE_RMBG_URL=https://rmbg.mycompany.internal/remove
export PROMPT_TO_BUNDLE_RECRAFT_VECTORIZE_URL=https://api.recraft.ai/v1/vectorize
export PROMPT_TO_BUNDLE_UPSCALER_URL=https://upscaler.modal.run/run
export PROMPT_TO_BUNDLE_VLM_URL=https://vlm-judge.internal/score

The 11 MCP tools

See the homepage tool section for the rundown. Full input/output schemas:

// asset_enhance_prompt (readOnly, idempotent, no network)
Input  : { brief: string, asset_type?, target_model?, brand_bundle?, transparent?, vector?, text_content? }
Output : AssetSpec { asset_type, rewritten_prompt, target_model, fallback_models[],
                     params {seed, negative_prompt?}, postprocess[], dimensions, safe_zone,
                     transparency_required, vector_required, text_content, warnings[] }

// asset_generate_logo / app_icon / favicon / og_image / illustration
Input  : { brief, brand_bundle?, output_dir?, + asset-specific options }
Output : AssetBundle { asset_type, brief, brand_bundle_hash, variants[], provenance, validations, warnings[] }

// asset_remove_background / vectorize / upscale_refine
Input  : { image: path, mode: enum, output_dir? }
Output : { output_path, method_used, warnings[] }

// asset_validate
Input  : { image, asset_type, brand_bundle?, intended_text?, run_vlm? }
Output : ValidationResult { pass, tier0{...}, tier1?{...}, tier2?{score, criteria, feedback}, warnings[] }

// asset_brand_bundle_parse
Input  : { source: path | raw text }
Output : { bundle: BrandBundle, hash, warnings[] }

Validation tiers

Tier 0 — deterministic (always runs): dimensions exact match, alpha channel presence where required, no checkerboard FFT signature, subject tight-bbox inside platform safe zone, file-size budget, JPEG-in-PNG DCT rejection.

Tier 1 — metrics (on first of a set or on request): VQAScore, palette ΔE2000, OCR Levenshtein for wordmarks, WCAG AA contrast at 16×16 for favicons.

Tier 2 — VLM-as-judge (opt-in, ~$0.003/call): 5–7 hard criteria per asset type, pairwise N-best ranking, failure feedback fed into regenerate loop.

FAQ

Why does Gemini return a weird checkerboard for transparent logos?

Its VAE is RGB-only. The model renders Photoshop's transparency-display pattern as RGB pixels because its training data contains screenshots with that visual. It is an architectural limit, not a prompt problem. The router will never send transparency-required briefs to Imagen or Gemini. See research 13.

Why can't I just ask for an SVG logo?

No raster diffusion model emits real SVG. Three paths work: (a) Recraft V3's native SVG output, (b) an LLM authors SVG for simple geometry, (c) generate raster → BiRefNet matte → K-means 6-color → vtracer → SVGO. The plugin does all three; the router picks based on brief complexity.

Why do my generated wordmarks come out garbled?

Every diffusion sampler garbles text past ~3 words; Ideogram 3 and gpt-image-1 are the best, but still unreliable past 5–8 words. The plugin drops wordmark text from the prompt past 3 words and composites real SVG/Canvas typography in the application layer instead.

Does this work offline / without API keys?

The pure-logic layer (classifier, router, rewriter, brand parser, validator, sync scripts) runs fully offline. Generation tools require at least one provider API key. Set PROMPT_TO_BUNDLE_DRY_RUN=1 to return dummy images instead of failing, useful for CI.

How is this different from appicon.co?

appicon.co takes a pre-made icon and fans it out. prompt-to-asset generates the mark too (and validates it, and vectorizes it, and exports a full platform bundle including Android adaptive + monochrome + PWA maskable + iOS 18 dark/tinted). Plus it's open source, MCP-native, and lives inside your editor.
