# Hybrid Vector Pipelines: Combining AI Raster, AI Vector, and Programmatic SVG

## Executive Summary

No single generator wins at every class of vector asset. The production-grade answer in 2025-2026 is a **router** that inspects the asset request (type, complexity, text content, brand strictness, output format) and dispatches to one of several pipelines:

1. **LLM-authored SVG** (Claude, GPT-5, Gemini) — best for simple geometric logos, flat icons, and app-icon glyphs where the shape can be described in <30 paths. Near-zero marginal cost; fully text-editable; survives arbitrary resize.
2. **Native vector diffusion / commercial vector model** (Recraft V4 SVG / V4 Pro SVG, SVGFusion, IconShop) — best for illustrative spots, stickers, and mid-complexity vector art that exceeds what an LLM can "draw" in code but still needs clean editable paths.

> **Updated 2026-04-21:** Recraft **V4** (including V4 Vector and V4 Pro Vector) was released in **February 2026** and is the current production model. Recraft V3 (released October 2024) is still accessible via API but V4 is now the recommended default. Pricing changed substantially: V4 Vector is **$0.08/image**, V4 Pro Vector is **$0.30/image** (vs the previously documented ~$0.01/call for V3 vectorize). Figma's native **Vectorize** tool launched **February 2026** (Full-seat users on Pro/Org/Enterprise plans with AI enabled) — it is an AI raster-to-vector converter integrated directly on the Figma canvas, directly competitive with the vtracer/Recraft workflow for designer-in-Figma contexts.
3. **AI raster → vectorizer** (Flux / SDXL / Nano Banana → vtracer / potrace / vectorizer.ai) — best when the source model has the right aesthetic (e.g. flat 2D illustration, isometric, line art) but the target format is SVG. Path count is the quality signal.
4. **Raster-only (skip SVG entirely)** — the correct decision for photoreal heroes, OG images, splash screens, any asset bound for PNG/JPEG anyway.
5. **Human-in-the-loop fallback** — route failed auto-vectorizations to `vectorizer.ai` (paid credits) or Recraft's "vectorize" endpoint; keep a queue for designer touch-up only when quality gates fail twice.

Three findings dominate:

1. **Path count is the single most useful cheap quality gate.** Good logos/icons generated by LLMs or native vector models tend to have <40 paths; vtracer output on clean flat illustrations sits at 80-400 paths; anything >1500 paths is almost always a failed vectorization of a photoreal image and should be auto-rejected.
2. **The "route by asset class" decision tree beats any single model.** Public behaviour of Canva Magic Studio and Figma's Vectorize tool confirms this: they do not use one model, they triage raster vs vector vs layer-extraction per input. The same pattern applies to an asset-generation backend.
3. **Caching by `sha256(prompt + model + params)` plus **seed reuse for variants** recovers 60-85% of spend** in typical production distributions where the same prompts are regenerated repeatedly (e.g. "transparent logo, blue, flat" across users and sessions). This aligns with OpenAI/Anthropic prompt-cache economics and with ComfyUI/Diffusers VAE-latent reuse.

## Decision Tree — Asset Type → Best Route

```text
request arrives
│
├── Is target format fundamentally raster (OG image, splash,
│   photoreal hero, marketing background)?
│   └── YES → Pipeline D: raster-only (Flux / Nano Banana / gpt-image-1)
│                          → optional rembg for alpha → PNG
│
├── Does the asset need crisp scaling to arbitrary sizes
│   (favicon 16px → 1024px app icon, print logo, UI icon set)?
│   └── YES → continue vector routing
│
├── Is the concept describable as <30 geometric paths?
│   (monogram, wordmark, flat glyph icon, simple mark)
│   └── YES → Pipeline A: LLM-authored SVG
│             (Claude 4 / GPT-5 / Gemini 2.5 Pro)
│             with constraint prompt + SVGO + render-check
│
├── Does it need illustrated character, sticker, spot illo,
│   or mid-complexity vector art with shading/gradients?
│   └── YES → Pipeline B: native vector model
│             (Recraft V4 SVG / V4 Pro SVG) — first choice
│             fallback: SVGFusion / IconShop / VectorFusion self-hosted
│
├── Does the brand already have a strong raster aesthetic
│   (e.g. a Flux LoRA, IP-Adapter style ref) that must be matched?
│   └── YES → Pipeline C: AI raster → vectorize
│             (Flux/SDXL + style LoRA → vtracer → SVGO)
│
└── Ambiguous? → Fan out A+B in parallel, score, keep best.
```

Operationalised as a lookup table:

| Asset Class            | Primary Route            | Secondary / Fallback           | Notes                                         |
|------------------------|--------------------------|--------------------------------|-----------------------------------------------|
| Wordmark / monogram    | A (LLM-SVG)              | B (Recraft V4 SVG)             | LLMs excel at typographic geometry.           |
| Flat glyph app icon    | A → B                    | C (Flux + vtracer)             | Validate rendered appearance at 16/32/64 px.  |
| Mascot / character     | B (Recraft)              | C (Flux+vtracer)               | LLM-SVG rarely captures organic form.         |
| Spot illustration      | B                        | C                              | Recraft V4 Pro SVG is current SOTA.           |
| Line-art icon set      | A (parallel 3-5 concepts)| B                              | Per Jeremy Watt's Claude-logo skill pattern.  |
| Isometric scene        | C (Flux+vtracer)         | B                              | Raster diffusion has stronger 3D priors.      |
| Favicon                | A                        | Down-sample chosen logo        | Re-author, don't resize a bitmap.             |
| OG / social share      | D (raster)               | —                              | SVG would be wasted; PNG/WebP is target.      |
| Splash / hero          | D                        | —                              | Photoreal → skip vector entirely.             |
| Emoji / sticker        | B                        | human-in-the-loop              | Vectorizer.ai for edge cases.                 |

## Pipeline Blueprints

Three reference architectures that an open-source asset-generation web app can wire directly.

### Blueprint 1 — "LLM-first SVG" (lowest cost, highest iteration speed)

Best for: icons, wordmarks, simple logos, favicon sets.

```
         ┌─────────────────────┐
user ──▶ │ Prompt Enhancer LLM │ (decomposes brief; injects
         │  (Claude / GPT-5)   │  constraints: ≤3 colors,
         └──────────┬──────────┘  viewBox, no fine detail)
                    │
                    ▼
         ┌─────────────────────┐
         │  Parallel generate  │  n=3–5 SVG candidates
         │  (same model, diff  │  (Claude Skills pattern:
         │  temperatures)      │  concepts.sh)
         └──────────┬──────────┘
                    │
          ┌─────────┴──────────┐
          ▼                    ▼
┌─────────────────┐    ┌──────────────────┐
│ SVG Linter      │    │ Headless render  │
│ - parse paths   │    │ (resvg / Playwr.)│
│ - count nodes   │    │ at 16, 32, 64,   │
│ - SVGO pass     │    │ 256, 1024 px     │
└────────┬────────┘    └────────┬─────────┘
         │                      │
         └──────────┬───────────┘
                    ▼
         ┌─────────────────────┐
         │ Scorer (LLM as      │  Gate: reject if
         │ judge + CLIPScore)  │  paths>60 or
         │                     │  render deltas >τ
         └──────────┬──────────┘
                    ▼
              best SVG → CDN
```

Key implementation notes drawn from public write-ups:

- **Claude Code "SVG logo designer" skill** (Jeremy Watt, neonwatty.com) demonstrates the parallel-concepts + HTML-preview ramp (16/32/64 px) pattern. Typical projects converge in 6-17 iterations with ≤5 paths per concept — empirical evidence that constraint-first LLM prompting yields small, robust SVGs.
- **OpenData Labs** documented using Claude "as a logo agency": the LLM acts as constraint-satisfier (single glyph, 2-3 colors, strong silhouette) rather than open-ended artist.
- Stack: Next.js app → Vercel AI SDK `streamText` against Claude/GPT → SVGO (`preset-default` minus `removeViewBox`) → resvg-js render verification → S3/CDN. The Vercel AI SDK `Image` element and `streamText` pattern works identically for streaming SVG text back to the browser.

### Blueprint 2 — "Native vector + raster sanity-check" (Recraft-centric)

Best for: spot illustrations, sticker packs, mid-complexity brand art.

```
 prompt
   │
   ▼
┌─────────────────────┐
│ Recraft API         │── returns .svg
│ /v1/images/gener…   │   (V4 SVG or V4 Pro SVG)
│ style=vector_illust │
└────────┬────────────┘
         │
         ├──▶ SVG → SVGO → path-count & bbox checks
         │
         ├──▶ resvg render at 512 px
         │        │
         │        ▼
         │   ┌──────────────────┐
         │   │ CLIP similarity  │  score render vs prompt
         │   │ (open_clip)      │
         │   └────────┬─────────┘
         │            │ pass?
         └────────────┤
                      │
                   yes│          │no
                      ▼          ▼
                    accept   ┌────────────────────┐
                             │ Fallback: vector-  │
                             │ize existing raster │
                             │ via Recraft        │
                             │ /vectorize or      │
                             │ vectorizer.ai API  │
                             └────────────────────┘
```

- Recraft V4 SVG and V4 Pro SVG (Replicate listings, Recraft API docs, Feb 2026 release) return **native editable SVG**, not traced bitmaps. Prompts up to 10,000 chars; ratios 1:1, 4:3, 3:2, 16:9, 9:16.
- Recraft also exposes a **/vectorize** endpoint — this is a clean "fallback to a paid SOTA vectorizer" when self-hosted vtracer produces junk. vectorizer.ai is the other market-standard option; both are safe commercial escape hatches.
- For a fully open-source stack, the substitute for Recraft is either **SVGFusion** (arXiv:2412.10437 — VP-VAE + VS-DiT, text-conditioned in vector latent space, avoids slow SDS) or **IconShop** (kingnobro/IconShop, autoregressive transformer over SVG command sequences — strong for monochrome icons). Both need GPUs and are heavier to operate than Recraft's API.

> **Updated 2026-04-21:** As of April 2026, SVGFusion's code and model weights are **not yet publicly released** — the project page (ximinng.github.io/SVGFusionProject/) lists them as "coming soon." The SVGX-SFT-1M training dataset is available on HuggingFace, but the model itself is not yet runnable without the released weights. Treat SVGFusion as a research reference rather than a production OSS option until the code drops.

### Blueprint 3 — "Flux raster + vtracer" (brand-tunable, fully open)

Best for: illustrative styles that must match a LoRA/IP-Adapter aesthetic; any scenario where the team already controls a Flux/SDXL pipeline.

```
 prompt + optional style ref
   │
   ▼
┌─────────────────────┐
│ Prompt Enhancer     │ injects: "flat vector illustration,
│                     │ thick outlines, 4-color palette,
│                     │ plain white background"
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ Flux.1 [schnell] or │
│ SDXL + style LoRA   │ → 1024×1024 PNG
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ rembg / BiRefNet    │ (optional) white-bg → alpha
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ Color quantize      │ k=4..8 palette reduction
│ (PIL / pyvips)      │
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ vtracer             │ hierarchical clustering → path
│ (color mode)        │ walking → simplify → curve fit
└────────┬────────────┘
         ▼
┌─────────────────────┐
│ SVGO + manual       │ remove micro-paths, merge
│ post-processing     │ same-fill groups, clamp precision
└────────┬────────────┘
         ▼
       SVG
```

This is exactly the pattern published in the llmkube "text-to-SVG in a weekend" walkthrough (`vecsmith`): local LLM prompt enhancement → Flux.1-schnell raster → vtracer → SVGO. The same pattern is the open-source spiritual sibling of Recraft's internal pipeline: hierarchical color clustering + spline fitting is how you get editable SVG out of a non-vector model.

Key insight: **prompt engineering on the raster side determines vector quality.** Phrases like "flat vector illustration, bold outlines, limited palette, plain background" collapse detail enough that vtracer produces O(100) paths instead of O(10,000). This is why Pipeline C must own the prompt template, not just accept the user's words.

Repo references that implement variants of the whole chain:

- **visioncortex/vtracer** — linear-time color raster→SVG (O(n) vs potrace O(n²)). Ships CLI + library.
- **ximinng/VectorFusion-pytorch** — SD + LIVE + DiffVG + SDS, open implementation of VectorFusion (CVPR 2023).
- **ximinng/SVGDreamer** — CVPR 2024, particle-based SDS for diverse SVGs.
- **kingnobro/IconShop** — autoregressive SVG transformer.
- **danielgatis/rembg**, **ZhengPeng7/BiRefNet** — background removal stages.
- **svg/svgo** — post-processing / optimization.
- **vercel/ai** examples and `examples.vercel.com/templates/next.js/ai-sdk-image-generator` — reference Next.js wiring for streaming generation UIs.

## Quality Gates

Cheap, automatable checks that catch the common failure modes before a designer sees them.

1. **Parseability**: round-trip the SVG through `svgo` / `xmldom`. Any parser exception → reject. Catches Recraft/LLM output with malformed path data.
2. **Path count bands** (the most useful single signal):
   - `paths < 5` → suspicious (blank or degenerate) → regenerate.
   - `5 ≤ paths ≤ 60` → typical for logos/icons → accept on other gates.
   - `60 < paths ≤ 600` → typical for illustrations / vtracer output → accept with render check.
   - `paths > 1500` → almost always failed vectorization of a photoreal/gradient-heavy raster → auto-route to fallback or switch to Pipeline D.
3. **Node-count-to-unique-color ratio**: many paths with the same fill = mergeable noise. Use SVGO's `mergePaths` and re-measure; if still high, reject.
4. **Render deltas across sizes**: rasterize at 16, 64, 256, 1024 px with resvg. Compute SSIM/LPIPS between the 1024 downsample and the native 16/64 renders. Large deltas at small sizes reveal hairline paths, sub-pixel strokes, or detail that vanishes — the icon-at-favicon-size failure mode.
5. **Bounding-box sanity**: non-trivial content must fill >30% of the viewBox; many failed generations place a tiny shape in a huge canvas.
6. **Text safety**: flag any `<text>` element. Text in SVG is rarely what the user wants for cross-platform logos; recommend converting to outlines or re-render with Ideogram / Recraft (strong typographic priors) if the brief requires text.
7. **CLIPScore / LLM-judge on rasterized SVG**: same scorer used for raster pipelines, applied to the rendered PNG. Bridges the "is this the right concept" question that geometry gates cannot answer.
8. **Accessibility pass (optional)**: add `role="img"` and `<title>` automatically during post-processing.

Failure routing:

- Two consecutive failures on Pipeline A → escalate to Pipeline B.
- Two failures on Pipeline B → fall back to Pipeline C (raster+vtracer).
- Two failures on C → queue for vectorizer.ai / Recraft /vectorize (paid) or human.

## Cost / Caching Strategy

Three layers of savings, compounding.

1. **Prompt-hash result cache.** Canonicalize `{model, prompt_text, params}` → SHA-256 → look up in KV (Redis / Cloudflare KV). Industry writeups report 15-30% duplicate LLM requests in production; for public asset generators the duplication rate is higher because prompts cluster on common briefs ("transparent logo for note-taking app", "minimal flat icon of a rocket"). Caching the full SVG artifact behind this key is near-free and returns in tens of milliseconds.
2. **Provider-side prompt caching** (Anthropic / OpenAI / Gemini). For LLM-SVG (Pipeline A), the system prompt is large (constraints, style guide, examples). Anthropic cached tokens are 90% cheaper; OpenAI 50%; Gemini 75%. In typical production systems 80-95% of input tokens repeat — aggregate input savings of 60-85% on Pipeline A. No code change required beyond `cache_control` on the system message.
3. **Seed + base-generation reuse for variants.** When the user asks for "the same logo in red", do not regenerate. Patterns:
   - **LLM-SVG**: re-emit with a single `fill="#RR.."` substitution; zero new tokens if done in post-processing.
   - **Recraft / SVG models**: call with the same seed + minimal prompt delta; Recraft and most diffusion providers respect seeds.
   - **Flux raster**: cache VAE latents of the base image; reuse for color/variant passes. ComfyUI/Diffusers workflows report 30-40% speedups from VAE-latent caching, and ~43% from text-embedding caching when prompts repeat.
4. **Asset-level dedup.** Hash the rendered PNG (perceptual hash, pHash) across all generated assets. Two distinct prompts often converge on visually identical logos; serve the existing artifact and record the new prompt as an alias.
5. **Tiered model routing.** Default to the cheapest model that satisfies the asset class. LLM-SVG (Pipeline A) is ~$0.003/request at current Claude 4 / GPT-5 rates; Recraft V4 Vector is ~$0.08/image (V4 Pro Vector ~$0.30); Flux.1-schnell on Replicate is ~$0.003/image but adds vtracer/SVGO infra. Price and quality are not monotonic — only escalate when quality gates fail.

> **Updated 2026-04-21:** The "$0.04/image" figure cited for Recraft V4 SVG was the V4 *raster* price. Recraft V4 Vector (native SVG output) is $0.08/image; V4 Pro Vector is $0.30/image. Adjust cost models accordingly — Pipeline B is now ~$0.08–$0.30 per asset, not ~$0.04.

Sample cache key:

```python
key = sha256(
    json_stable({
        "model": "claude-4-sonnet",
        "system_hash": sha256(SYSTEM_PROMPT),
        "prompt": user_prompt_normalized,
        "params": {"temperature": 0.7, "max_tokens": 4096},
        "pipeline": "A",
        "output_format": "svg",
    })
).hexdigest()
```

Normalizing the user prompt (lower-case, whitespace-collapse, strip emoji) is what pushes the cache-hit rate from "low" to "useful". Follow the SHA-256 canonical-serialization discipline Gaurav Dagde wrote up (dev.to, "Prompt Hashing for Duplicate Detection") — include the model and params, not just text.

## Commercial Case Studies (public info only)

- **Canva Magic Studio / Magic Layers**: public product pages and Alibaba's side-by-side analysis describe a proprietary "Design Model" that **ingests raster** (PNG/JPG), then separates typography / graphics / layout into editable layers. The internal output is vector-like editable components, but upstream generation leans raster; vectorization is a post-hoc analysis step. Implication: Canva routes raster-first, then lifts structure — opposite of Recraft's vector-first stance.
- **Figma Vectorize tool** (launched **February 2026**, not mid-2025 as previously noted): raster → vector directly on canvas, one-click conversion of placed raster images to editable vectors. Available to Full-seat users on Professional, Organization, and Enterprise plans with AI enabled; also available in Figma Slides and Buzz. Uses AI credits. Figma's plugin-model architecture forces outputs to play with variables, components, and auto-layout, so any AI that generates inside Figma is implicitly producing vector. The Vectorize tool is an AI raster-to-vector converter — analogous to vtracer with ML post-processing, now integrated into the core canvas workflow.

> **Updated 2026-04-21:** Previous "mid-2025" launch date was incorrect. Figma Vectorize launched February 2026 per Figma's official blog post ("For the Love of Craft: Vectorize Images in Figma Design and Draw").
- **Gamma**: image options are configurable per deck (`imageOptions.source`: `aiGenerated`, `webFreeToUseCommercially`, `noImages`). Public docs do not describe a dedicated vector pipeline; presentations export as PDF/PPTX/PNG, so Gamma's routing is effectively "raster for illustrations, native shapes for diagrams".

The common thread: **every commercial tool operates a router, not a model**. The open-source blueprint should mirror this.
